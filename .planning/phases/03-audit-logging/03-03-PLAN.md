  <behavior>
    get_audit_log behavior:
    - Calls middleware::check_permission(session, Resource::AuditLogs, Action::Read)
    - Returns Err(Unauthorized) for any role without AuditLogs:Read (NurseMa, BillingStaff, FrontDesk)
    - Provider: calls read_audit_entries(&conn, Some(&user_id), limit, offset) — own entries only
    - SystemAdmin: calls read_audit_entries(&conn, None, limit, offset) — all entries
    - Returns Vec<AuditLogRow> serialized as camelCase JSON

    verify_audit_chain behavior:
    - Calls middleware::check_permission(session, Resource::AuditLogs, Action::Read)
    - Returns Err(Unauthorized) for roles without AuditLogs:Read (same as above)
    - Calls audit::query::verify_audit_chain(&conn)
    - Returns ChainVerificationResult { intact: bool, entry_count: i64 }

    Do NOT write an audit entry for get_audit_log or verify_audit_chain invocations
    (per pitfall 4 in research: audit log reads do not themselves generate audit entries).
  </behavior>
  <action>
    Create src-tauri/src/commands/audit.rs:

    ```rust
    use serde::Serialize;
    use tauri::State;
    use crate::audit::{entry::DeviceId, query};
    use crate::auth::session::SessionManager;
    use crate::db::connection::Database;
    use crate::error::AppError;
    use crate::rbac::{middleware, roles::{Action, Resource, Role}};

    #[derive(Debug, Serialize)]
    #[serde(rename_all = "camelCase")]
    pub struct ChainVerificationResult {
        pub intact: bool,
        pub entry_count: i64,
    }

    #[tauri::command]
    pub fn get_audit_log(
        db: State<'_, Database>,
        session: State<'_, SessionManager>,
        limit: Option<i64>,
        offset: Option<i64>,
    ) -> Result<Vec<query::AuditLogRow>, AppError> {
        let (user_id, role) =
            middleware::check_permission(&session, Resource::AuditLogs, Action::Read)?;
        let conn = db.conn.lock()
            .map_err(|e| AppError::Database(e.to_string()))?;
        let user_filter = match role {
            Role::Provider => Some(user_id.as_str()),
            Role::SystemAdmin => None,
            _ => return Err(AppError::Unauthorized("No audit log access".to_string())),
        };
        // Note: user_filter borrows user_id; use as_str() within the same scope
        query::read_audit_entries(&conn, user_filter, limit.unwrap_or(100), offset.unwrap_or(0))
    }

    #[tauri::command]
    pub fn verify_audit_chain(
        db: State<'_, Database>,
        session: State<'_, SessionManager>,
    ) -> Result<ChainVerificationResult, AppError> {
        middleware::check_permission(&session, Resource::AuditLogs, Action::Read)?;
        let conn = db.conn.lock()
            .map_err(|e| AppError::Database(e.to_string()))?;
        let intact = query::verify_audit_chain(&conn)?;
        let entry_count: i64 = conn.query_row(
            "SELECT COUNT(*) FROM audit_logs", [], |row| row.get(0)
        ).unwrap_or(0);
        Ok(ChainVerificationResult { intact, entry_count })
    }
    ```

    Note on borrowing: `user_id` is a String from check_permission. For the Provider case,
    `user_filter = Some(user_id.as_str())` — ensure user_id lives long enough for the
    read_audit_entries call (it does, both are in the same function scope).

    Modify src-tauri/src/commands/mod.rs: add `pub mod audit;`
