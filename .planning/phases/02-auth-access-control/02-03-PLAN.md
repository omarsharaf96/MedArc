<objective>
Add TOTP-based multi-factor authentication and Touch ID biometric support for session unlock.

Purpose: MFA is a HIPAA security best practice for ePHI access. TOTP provides a second authentication factor during login, while Touch ID provides convenient biometric unlock after session lock (not replacing password for initial login, per Pitfall 7 in RESEARCH.md).

Output: TOTP module with secret generation/QR codes/verification, Touch ID integration module, and Tauri commands for MFA enrollment and biometric authentication.
</objective>

  <action>
    1. Create src-tauri/src/commands/mfa.rs with Tauri commands:
       - `setup_totp(db: State<Database>, session: State<SessionManager>)`:
         Require active session. Get current user. Generate TOTP setup via auth::totp::generate_totp_setup(username). Return TotpSetup (secret_base32, otpauth_url, qr_base64) to frontend for QR display. Do NOT store secret in DB yet -- wait for verification.
       - `verify_totp_setup(db: State<Database>, session: State<SessionManager>, secret_base32: String, code: String)`:
         Require active session. Verify the code against the provided secret. If valid, UPDATE users SET totp_secret = secret_base32, totp_enabled = 1 WHERE id = user_id. Return Ok("TOTP enabled"). If invalid, return Err -- user must scan QR and enter correct code to confirm setup.
       - `disable_totp(db: State<Database>, session: State<SessionManager>, password: String)`:
         Require active session. Verify password (security measure). UPDATE users SET totp_secret = NULL, totp_enabled = 0. Return Ok(()).
       - `check_totp(db: State<Database>, session: State<SessionManager>, code: String)`:
         Get user's totp_secret from DB. If totp_enabled is false, return Err("TOTP not enabled"). Verify code. Return Ok(valid: bool). This is called during login flow when MFA is required.
       - `check_biometric(session: State<SessionManager>)`:
         Return { available: auth::biometric::check_biometric_available(), enabled: current_user.touch_id_enabled }.
       - `enable_touch_id(db: State<Database>, session: State<SessionManager>, password: String)`:
         Require active session. Verify password. UPDATE users SET touch_id_enabled = 1. Return Ok(()).
       - `disable_touch_id(db: State<Database>, session: State<SessionManager>)`:
         Require active session. UPDATE users SET touch_id_enabled = 0. Return Ok(()).

    2. Update src-tauri/src/commands/mod.rs to declare mfa module.

       - If using tauri-plugin-biometry, add `.plugin(tauri_plugin_biometry::init())` to the Tauri builder chain (before .invoke_handler). If the plugin fails to compile, skip this step.
       - Register all MFA commands in generate_handler: commands::mfa::setup_totp, commands::mfa::verify_totp_setup, commands::mfa::disable_totp, commands::mfa::check_totp, commands::mfa::check_biometric, commands::mfa::enable_touch_id, commands::mfa::disable_touch_id.
