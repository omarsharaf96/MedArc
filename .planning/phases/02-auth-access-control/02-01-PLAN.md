<objective>
Build the authentication foundation: user account creation with Argon2id password hashing, login/logout flow, and a session state machine with configurable inactivity timeout.

Purpose: This is the security gate for the entire application. No other phase (audit logging, patient data, clinical docs) can function without authenticated users and sessions. This plan creates the backend auth infrastructure that all subsequent plans build on.

Output: Database schema for users/sessions/settings, Rust auth module (password hashing, session management), and Tauri commands for register/login/logout/session operations.
</objective>

  <action>
    1. Create src-tauri/src/commands/auth.rs with these Tauri commands:
       - `register_user(db: State<Database>, session: State<SessionManager>, username, password, display_name, role)`:
         Validate password >= 12 chars. Check session: only allow registration if (a) no users exist in DB (first-run bootstrap) or (b) current user is SystemAdmin. Check username uniqueness via DB query. Hash password with auth::password::hash_password. INSERT into users table. Return UserResponse. Do NOT include password_hash in response.
       - `login(db: State<Database>, session: State<SessionManager>, username, password)`:
         SELECT user by username. If not found, return Authentication error (generic "Invalid credentials" -- don't reveal which field). Check is_active. Check locked_until for account lockout. Verify password with auth::password::verify. Reset failed_login_attempts on success. On failure, increment failed_login_attempts; if >= max_failed_logins (from app_settings), set locked_until. Call session.login(user_id, role). INSERT session row into sessions table. Return UserResponse + SessionInfo.
       - `logout(session: State<SessionManager>)`:
         Call session.logout(). UPDATE session row state to 'expired'. Return Ok(()).

    2. Create src-tauri/src/commands/session.rs with these Tauri commands:
       - `lock_session(session: State<SessionManager>)`: Call session.lock(). UPDATE session row state to 'locked'.
       - `unlock_session(db: State<Database>, session: State<SessionManager>, password: String)`:
         Get current user from session (must be Locked state). Verify password against stored hash. If valid, call session.unlock(). UPDATE session row state to 'active'. If invalid, return Authentication error.
       - `refresh_session(session: State<SessionManager>)`: Call session.refresh_activity(). UPDATE session last_activity.
       - `get_session_state(session: State<SessionManager>)`: Return current SessionInfo (state, user_id, role). This is how the frontend knows if user is logged in, locked, etc.
       - `get_session_timeout(db: State<Database>)`: Read session_timeout_minutes from app_settings. Return u32.
