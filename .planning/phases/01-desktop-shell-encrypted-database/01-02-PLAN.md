    **FHIR commands (src-tauri/src/commands/fhir.rs):**
    Implement five Tauri commands, each taking `State<Database>`:

    1. `create_resource(db, input: CreateFhirResource) -> Result<FhirResource, AppError>`:
       - Generate UUID via `uuid::Uuid::new_v4()`
       - Set `last_updated` to current UTC ISO 8601 via `chrono::Utc::now().to_rfc3339()`
       - INSERT into fhir_resources
       - Return the created FhirResource

    2. `get_resource(db, id: String) -> Result<FhirResource, AppError>`:
       - SELECT by id from fhir_resources
       - Parse JSON column back to serde_json::Value
       - Return AppError if not found (use a NotFound variant or Database error with descriptive message)

    3. `list_resources(db, resource_type: Option<String>) -> Result<FhirResourceList, AppError>`:
       - If resource_type provided: SELECT WHERE resource_type = ?
       - If None: SELECT all
       - Return list with total count
       - Order by last_updated DESC

    4. `update_resource(db, input: UpdateFhirResource) -> Result<FhirResource, AppError>`:
       - SELECT current version_id for optimistic locking
       - UPDATE resource JSON, increment version_id, set last_updated and updated_at
       - Return error if id not found
       - Return the updated FhirResource

    5. `delete_resource(db, id: String) -> Result<(), AppError>`:
       - DELETE from fhir_resources WHERE id = ?
       - Return error if no rows affected (not found)
       - CASCADE will handle fhir_identifiers cleanup
