  <action>
    **Keychain module (src-tauri/src/keychain.rs):**
    - Implement `get_or_create_db_key() -> Result<String, AppError>`:
      - Use `keyring::Entry::new("com.medarc.emr", "database-encryption-key")`
      - On `Ok(key)`: return existing key
      - On `Err(keyring::Error::NoEntry)`: generate new key via `generate_random_key()`, store it with `entry.set_password()`, return it
      - On other errors: return `AppError::Keychain`
    - Implement `generate_random_key() -> String`:
      - Generate 32 bytes using `getrandom::fill()`
      - Format as SQLCipher raw hex key: `format!("x'{}'", hex::encode(key_bytes))`
      - Raw keys skip PBKDF2 entirely (per research recommendation for fast startup)

    **Database connection (src-tauri/src/db/connection.rs):**
    - Define `pub struct Database { pub conn: Mutex<Connection> }`
    - Implement `Database::open(db_path: &str, key: &str) -> Result<Self, AppError>`:
      - `Connection::open(db_path)`
      - CRITICAL: `conn.pragma_update(None, "key", key)` MUST be the FIRST statement after open. No exceptions.
      - Verify encryption: `conn.pragma_query_value(None, "cipher_version", |row| row.get::<_, String>(0))` -- if this fails, the key is wrong
      - Enable WAL mode: `conn.pragma_update(None, "journal_mode", "WAL")`
      - Enable foreign keys: `conn.pragma_update(None, "foreign_keys", "ON")`
      - Return `Database { conn: Mutex::new(conn) }`

    **Database module (src-tauri/src/db/mod.rs):**
    - Re-export `connection` and `migrations` modules.

    **Migrations (src-tauri/src/db/migrations.rs):**
    - Define migrations using `rusqlite_migration::Migrations::from_slice`. For now, include one initial migration that creates a simple `app_metadata` table:
      ```sql
      CREATE TABLE IF NOT EXISTS app_metadata (
          key TEXT PRIMARY KEY NOT NULL,
          value TEXT NOT NULL,
          updated_at TEXT NOT NULL DEFAULT (datetime('now'))
      );
      INSERT OR IGNORE INTO app_metadata (key, value) VALUES ('schema_version', '1');
      INSERT OR IGNORE INTO app_metadata (key, value) VALUES ('created_at', datetime('now'));
      ```
      This validates the migration system works. The FHIR resource tables will be added in Plan 02.
    - Implement `pub fn run(db: &Database) -> Result<(), AppError>` that acquires the mutex lock, calls `MIGRATIONS.to_latest(&mut conn)`.
    - Add a `#[cfg(test)]` test: `fn migrations_are_valid() { assert!(MIGRATIONS.validate().is_ok()); }`

    **Wire everything in lib.rs (src-tauri/src/lib.rs):**
    - Declare modules: `mod commands; mod db; mod error; mod keychain;`
    - Implement `pub fn run()`:
      - `tauri::Builder::default()`
      - `.setup(|app| { ... })`:
        1. Get app data dir: `app.path().app_data_dir()?`
        2. Create dir if not exists: `std::fs::create_dir_all(&app_data_dir)?`
        3. Build db path: `app_data_dir.join("medarc.db")`
        4. Get key: `keychain::get_or_create_db_key()?`
        5. Open database: `db::connection::Database::open(db_path_str, &key)?`
        6. Run migrations: `db::migrations::run(&database)?`
        7. Manage state: `app.manage(database);`
        8. Return `Ok(())`
      - `.invoke_handler(tauri::generate_handler![commands::health::check_db, commands::health::get_app_info])`
      - `.run(tauri::generate_context!()).expect("error while running tauri application")`

    Ensure proper use statements throughout. Run `cargo check` and `cargo test` to verify compilation and migration validation.
